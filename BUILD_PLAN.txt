Boss Assistant Build Plan

Goal
Build a personal AI assistant on macOS with a multi-provider model layer. The
system uses configurable model routing so any capable provider (OpenAI, Anthropic,
Ollama, or OpenAI-compatible endpoints) can serve a given role. Model names are
always configurable — no single provider is hardcoded as the permanent default.

Architecture
User Input
  -> General Agent (configurable, default cloud)
    -> Mac Agent (configurable + tools + MCP)
    -> Research Agent (configurable + web search)
    -> Reasoning Agent (configurable)
    -> Code Agent (configurable)
  -> Output + Memory Persistence

Provider routing is handled by `boss/providers/registry.py`. Each provider
registers capabilities (chat, streaming, vision, embeddings, tool_use) and the
registry resolves which provider handles each mode (chat, reasoning, embeddings,
vision) with a configurable fallback chain.

Completed Phases
Phase 0 — Environment setup (Homebrew, Python 3.12+, Node.js, venv, packages)
Phase 1 — Config-driven cloud model provider, agents, handoffs, CLI runner
Phase 2 — Native macOS function tools registered with the Mac agent
Phase 3 — MCP server definitions, Apple integration, hosted web search
Phase 4 — Input guardrails for dangerous requests
Phase 5 — Conversation persistence across restarts
Phase 6 — Streaming CLI with rich output
Phase 7 — Tool governance, permissions, memory subsystem, background jobs
Phase 8 — Code review, multi-agent workers, deploy/preview, loop engine
Phase 9 — Multi-provider registry (OpenAI + Ollama), codebase intelligence

Current Phase — Phase 10: Provider Expansion and Hardening

   - Replace allow_origins=["*"] with localhost + tauri origins

5. MCP server version pinning
   - Pin apple-mcp and @modelcontextprotocol packages to specific versions
   - Remove @latest tags

6. Tool parameter extraction error handling

7. Ollama as optional local provider (available, not default)

Verification Targets
- Basic Q and A routes through the general entry agent.
- Complex reasoning routes to the configured reasoning provider.
- Mac requests call macOS tools.
- Dangerous requests are blocked.
- Session history persists across restarts.
- Provider registry resolves correct provider per mode.
- Anthropic provider passes health check when API key is set.
- Capability fallback works when a provider is unavailable.
